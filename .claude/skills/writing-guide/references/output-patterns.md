# Output Format Patterns

Use these templates when formatting responses. Select the pattern that matches the current mode and intensity.

---

## Pattern A: Diagnostic Response
*Used in: scene review, character review, prose review, outline review*

```
DIAGNOSIS — [mode label]

[Opening: 1-2 sentences naming the core craft concern]

What's working:
  - [item]

What needs attention:
  - [item]: [specific explanation]

Craft principle at stake:
  [1-2 sentences referencing the relevant leaf guide concept]

Options:
  A. [Option]: [trade-off]
  B. [Option]: [trade-off]

Before you decide:
  [1-2 questions]
```

---

## Pattern B: Structural Report
*Used in: structural diagnosis, revision planning, retrospective audit at firm/diagnostic intensity*

```
STRUCTURAL REPORT — [mode label]
Intensity: [firm/diagnostic]
Guides consulted: [list]

Core problem:
  [Direct statement of the primary structural issue]

Contributing factors:
  1. [Factor]: [connection to core problem]
  2. [Factor]: [connection to core problem]

Scene-level evidence:
  [Specific beats or scenes demonstrating the problem]

Options:
  A. [Approach]: [what changes, what it costs, what it gains]
  B. [Approach]: [what changes, what it costs, what it gains]

Sequencing recommendation:
  [Revision order: what to fix first and why]

Open questions before acting:
  1. [Question]
  2. [Question]
  3. [Question]
```

## Pattern C: Checklist Run

```
CHECKLIST — [CHK file name]

Item 1: [Item text]
  Status: Pass / Concern / Unclear
  Notes: [brief explanation if not Pass]

[continue for all items]

Summary:
  Passed: N / Concerns: N / Unclear: N

Recommended next step: [one concrete action]
```

---

## Pattern D: Toll Gate

```
TOLL GATE: [Gate name]

[1-sentence reason this gate is firing]

Before proceeding, consider:
  - [Question 1]
  - [Question 2]
  - [Question 3]

[Checklist items inline if applicable]

Options:
  - [Primary option]
  - [Secondary option]
  - [Bypass option, if available at current intensity]
```

---

## Pattern E: Ripple Review

See the **Ripple-Effect Review Protocol** section of SKILL.md for the full format. Quick reference:

```
RIPPLE REVIEW: [Change type] — [brief description]

Scope: [narrow / moderate / extensive] — [1-sentence justification]

First-order consequences:
  [Category]: [Specific element affected] — [how it is affected]

Second-order consequences:
  [Items that chain from first-order]

Open questions before this change is stable:
  1. [Question]
  ...

Continuity audit recommended: [yes/no]

Next step: [one concrete recommendation]
```

## Pattern F: Gentle Mode Short Response

```
[Opening: 1 sentence naming what's working]

One thing worth looking at: [single craft concern, framed as a question]

[Optional: 1 relevant craft principle in plain language]

What's your instinct here — [open question]?
```

---

## Pattern G: Ghostwriting Safeguard

```
MENTOR BOUNDARY

[writing-guide] does not write prose, dialogue, or story content. Its role is to
help you think through, diagnose, and decide — not to generate.

What I can do instead:
  - Diagnose why this [scene/passage/character beat] isn't working
  - Walk through craft principles that apply to this situation
  - Ask questions that will help you find your own solution
  - Compare structural options and their trade-offs
  - Run a checklist to identify what's missing

What would be most useful right now?
```

In `gentle` mode only, add: "This isn't a judgment about your writing — it's how I work best as a mentor."

## Pattern H: Retrospective Report

```
RETROSPECTIVE AUDIT — [project name/context if known]

Changes identified this draft phase:
  - [Change 1]: [type]
  - [Change 2]: [type]

Ripple assessment per change:
  [Change 1]:
    First-order: [items]
    Unresolved: [open questions]

Highest-risk unresolved items (ranked):
  1. [Item]: [why it's highest risk]
  2. [Item]

Recommended audit sequence:
  Step 1: [action]
  Step 2: [action]

Continuity review needed: [yes/no]
```
